Pourquoi utiliser un Framework ?

Car ils sont conçus pour aider les programmeurs dans leurs tâches. Tout particulièrement pour les Framework web dont font partis les Framework CSS qui sont les plus utilisés car ils présent une multitude d’intérêts.

Avantages

• Ils sont multiplateformes, c’est-à-dire que malgré des navigateurs différents le style sera quasi-identique. Les Framework sont dit cross-browser, c’est-à-dire que quelques soit le navigateur du client le rendu sera similaire et d’une compatibilité quasi-parfaite.
• Ils font gagner un temps considérable en proposant un « fondation » pour notre travail permettant d’avoir une base de départ solide.
• Ils normalisent énormément les affichages en offrant un assortiment homogène de styles.
• Ils gèrent en général la présentation en divisant la page avec une grille pour simplifier le positionnement des différentes sections du site.
• Ils offrent des fonctionnalités ajoutées tels que des barres de navigation, des boutons stylisés, etc.
• Ils permettent d'optenir des site web facilements maintenables.
• Enfin, et pas des moindres, ils sont très adaptés aux designs responsive pour les nouvelles interfaces mobiles principalement.

Inconvénients

• Prise en main, en effet pour dompter efficacement un Framework il est impératif de le connaître. Ce qui implique une aclimatation à l'environnement et donc un investissement de temps au préalable.
• Réputation, contrairement à ce que l’on peut penser les Framework sont « victimes » de leur succès. Par conséquent tous les sites qui utilisent des Framework ont une présentation très similaire ce qui peut lasser les utilisateurs et endigue la productivité et l'originalité des développeurs.
• Dépendance, une fois que l'on a adopter un framework on est dépendant de ce dernier, si il y a des mises à jour on est contraint d'adapter notre développement en conséquence.
